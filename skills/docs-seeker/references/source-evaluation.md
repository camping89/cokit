# Source Evaluation

How to evaluate documentation sources for reliability.

## Evaluation Criteria

### Authority
- Official documentation?
- Recognized expert author?
- Reputable organization?
- Community-verified?

### Currency
- Publication date visible?
- Last updated recently?
- Version matches yours?
- Active maintenance?

### Accuracy
- Code examples work?
- Information consistent?
- Cross-referenced?
- No obvious errors?

### Completeness
- Covers your use case?
- Explains edge cases?
- Includes examples?
- Links to related docs?

## Source Ranking

### Tier 1: Official Sources
- Official documentation
- API references
- Official tutorials
- Release notes

### Tier 2: Primary Sources
- GitHub repositories
- Author blogs
- Conference talks
- RFC documents

### Tier 3: Community Sources
- Stack Overflow (verified)
- Community tutorials
- Technical blogs
- Forum discussions

### Tier 4: Use with Caution
- Outdated articles
- Unverified answers
- AI-generated content
- Anonymous sources

## Red Flags

### Content Issues
- No date/version info
- Broken code examples
- Outdated dependencies
- Copy-pasted errors

### Source Issues
- Unknown author
- No references
- Clickbait title
- Excessive ads

## Validation Checklist

- [ ] Source is authoritative?
- [ ] Content is current?
- [ ] Version matches?
- [ ] Examples tested?
- [ ] Cross-referenced?
