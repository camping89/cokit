# AGENTS.md

## Project Overview

CoKit is a CLI tool that generates GitHub Copilot customizations from ClaudeKit content. The project follows the Awesome Copilot format specification.

## Repository Structure

```
.
├── agents/           # Custom agents (.agent.md)
├── prompts/          # Reusable prompts (.prompt.md)
├── instructions/     # Coding standards (.instructions.md)
├── skills/           # Agent skills with assets
├── collections/      # Curated collections (.collection.yml)
├── docs/             # Generated documentation
├── eng/              # Build scripts
├── src/              # CLI source code
├── templates/        # Project templates
└── bin/              # CLI entrypoint
```

## Development Workflow

### Building

```bash
npm run build          # Generate README files
```

### Validation

```bash
npm run collection:validate   # Validate collections
npm run skill:validate        # Validate skills
```

### Creating Resources

```bash
npm run collection:create -- --id <id>
npm run skill:create -- --name <name>
```

## File Formats

### Agent Files (*.agent.md)

```yaml

---
```

### Instruction Files (*.instructions.md)

```yaml

---
```

### Skill Files (skills/*/SKILL.md)

```yaml
---
name: 'skill-name'

---
```

- Use ES modules (.mjs for scripts)

- Single quotes in YAML frontmatter

# Migration Guide

## From Old CoKit (v1.x)

If you used the previous version of CoKit:

1. Remove old files:
   ```bash
   rm -rf .github/copilot-instructions.md
   rm -rf ~/.copilot/skills/
   ```

2. Run new setup:
   ```bash
   npx cokit init --all
   ```

## From Claude Code

### Key Differences

| Feature | Claude Code | CoKit/Copilot |
|---------|-------------|---------------|
| Commands | Slash commands with `$ARGUMENTS` | Prompts, type `/ck-*` |
| Agents | Subagents via Task tool | Single agent |
| Skills | Auto-loaded from ~/.claude/skills/ | Manual via instructions |
| Hooks | Pre/post hooks | Not supported |
| Context | Session state | Chat context only |

### Command Mapping

| Claude Code | CoKit |
|-------------|-------|
| `/fix <issue>` | `/ck-fix` then describe issue |
| `/plan <task>` | `/ck-plan` then describe task |
| `/code <plan>` | `/ck-code` then reference plan |
| `/test` | `/ck-test` |

### What Works the Same

- Core prompts have same intent
- Instructions apply automatically
- Skills provide specialized knowledge

### What's Different

1. **No Arguments**: Instead of `/fix $ARGUMENTS`, type `/ck-fix` then describe the issue in chat.

2. **No Subagents**: Claude Code delegates to specialized agents. Copilot uses a single agent with loaded instructions.

3. **No Hooks**: Claude Code supports pre/post hooks. Copilot uses static configuration only.

4. **No Session State**: Claude Code maintains state across commands. Copilot uses chat context.

### Tips for Transition

1. Be more explicit in chat since there's no argument passing
2. Reference files explicitly instead of relying on context
3. Use multiple prompts for complex workflows
4. Load relevant instructions before starting
```

# FAQ

## General

**Q: What is CoKit?**
A: CoKit generates GitHub Copilot customizations from ClaudeKit content.

**Q: Is CoKit free?**
A: Free for personal and educational use (CC BY-NC 4.0).

## Installation

**Q: Where are files installed?**
A: Project mode installs to `.github/`, global mode to `~/.copilot/`.

**Q: Can I use both project and global?**
A: Yes! Run `cokit init --all`.

## Usage

**Q: Why `/ck-` prefix?**
A: Prevents conflicts with other tools. Easy to identify CoKit prompts.

**Q: How do I pass arguments?**
A: Type the prompt first, then describe your request in chat.

## Troubleshooting

**Q: Prompts not showing?**
A: Restart VS Code after running `cokit init`.

**Q: Skills not working?**
A: Enable in VS Code: Settings > `github.copilot.chat.useAgentSkills`.

**Q: How do I update?**
A: Run `cokit update` or `npx cokit@latest init --overwrite`.
```
